# Plantex Sales Dashboard 🌿

A powerful, entirely Server-Side Rendered (SSR) analytics platform designed for detailed multi-metric visualization, and fast asynchronous execution. Plantex provides specialized data aggregations for Business, CEO, and Category Manager roles in real time!

## 🚀 Features

- **Full Data Integrity & Isolation:** Every user maintains localized datasets securely mapped using advanced M2M relationships. This guarantees high-level analytics isolated natively per user.
- **Asynchronous Data Pipelines:** Complex sales pipelines parsing multiple chunks of CSV sets are completely offloaded into background workers running **Celery** to ensure zero-stutter or connection-drops!
- **Real-Time Progress Tracking:** File parsing tasks effortlessly feed progress percentages directly back into the browser over WebSockets utilizing **Django Channels (Daphne)** combined with **Redis** ensuring seamless interaction.
- **Server-Side Rendered (SSR):** Context injection using Pandas and NumPy natively into Django templates keeping standard `Business Report`, `CEO Report`, and `Category Report` templates free of cluttered vanilla JavaScript processing logic!
- **Sleek Graphics Engine:** Fluid graph generations implemented purely utilizing Chart.js driven by localized REST contexts natively generated by the SSR engine!

---

## 🛠️ Technology Stack

- **Backend:** Django, Django REST Framework, Celery, Django Channels, Pandas, NumPy 
- **Broker & Backend Engine:** Redis Server
- **Frontend Layer:** HTML5, CSS3, Vanilla JS, Chart.js

## ⚙️ Installation & Usage

### 1. Prerequisites 

Make sure the following dependencies are installed globally on your machine:
- `python 3.x`
- `redis-server` (Running on default port `6379`)

### 2. Setup

Clone the project locally and jump into root directory. Setup your environment and load Django packages inside `pltx_dashboard`:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Run database migrations natively via the ORM:
```bash
python manage.py makemigrations
python manage.py migrate
```

### 3. Spin Up Servers 

To witness the full robust power of the WebSocket progress bars alongside the background queue you will need to open **two** terminal windows.

**Terminal 1 (Start Django Channels server using Daphne):**
```bash
cd pltx_dashboard
daphne pltx_dashboard.asgi:application --port 8000
```
> Note: Standard `runserver` will not run HTTP / WebSockets interchangeably as efficiently as Daphne. Daphne is natively configured within `asgi.py`. 

**Terminal 2 (Start Celery Worker):**
```bash
cd pltx_dashboard
celery -A pltx_dashboard worker -l info
```

Navigate to `http://127.0.0.1:8000/` and test real-time dashboard analytics out right now!
